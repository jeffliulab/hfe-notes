    HFE 笔记开发手册

    开发要点

    • 每页先回答三件事：这章讲什么、为什么重要、怎么用。
    • 页面主线必须足够清晰，学生在首屏 300-500 字内就要知道这页的核心问题。
    • 正文优先服务理解，原文追溯放在页尾折叠附录。
    • 图片只做正文解释，不单独堆成图廊。

    内容要点

    • 每页必须有 本页主问题。
    • 每页必须有 本章重点 或案例关键结论。
    • 每页必须有难点解释、至少一个案例、以及结尾复习总结。
    • 术语先给中文直觉，再给英文术语。
    • 模块之间必须有承接关系，不能变成散块堆砌。

    提示要点

    • tip：开头记忆锚点、结尾复习要点。
    • note：主问题、关键结论、方法定位。
    • warning：易混点、分析陷阱、常见误判。
    • example：案例、worked example、情景练习。

    开发方法

    1. 读源文件，先确认这页真正的主问题和课程位置。
    2. 先判断来源属于 ENP111 还是 ENP112，一级主题始终按知识源目录归类。
    3. 抽主线，确定这页属于 概念页 / 方法页 / 案例页 哪一类。
    4. 选配图，只保留能帮助解释正文的关键图。
    5. 先写中文讲义页，再补英文镜像页。
    6. 检查原文逐行覆盖仍然完整，附录映射不能丢。
    7. 运行 python scripts/build_hfe_notes.py 和 python -m mkdocs build 做最终验证。

    目录约定

    • 正式页面路径使用 docs/ENP111/ 或 docs/ENP112/
    • 旧的 docs/HFE_* 路径只保留兼容入口，不再作为正式页面位置

    页面骨架

    • 概念页： 本章重点 -> 先记住一句话 -> 概念是什么 -> 为什么重要 -> 关键框架/分类 -> 难点讲解 -> 案例 -> 这套概念怎么用 -> 本章总结
    • 方法页： 本章重点 -> 先记住方法定位 -> 这个方法解决什么问题 -> 输入与输出 -> 操作步骤 -> 最容易做错的地方 -> worked example -> 和前后方法的关系 -> 本章总结
    • 案例页： 案例关键结论 -> 先记住这个案例的一句话判断 -> 背景与 stakes -> 事件链/失误链 -> 用哪些课程概念分析 -> 系统层问题 -> 本章总结
